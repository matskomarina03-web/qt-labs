Тема проєкту:
        Застосунок дозволяє користувачу проходити логічну гру Nonogram (Picross) у форматі раундів із поступовим підвищенням складності, з таймером, перевіркою правильності та збереженням результатів кожного раунду.

Ідея геймплею (розширення логіки)

Застосунок побудований не як просто набір рівнів, а як система раундів:

        гра складається з послідовних раундів (наприклад 1–30)
        кожен раунд = одна головоломка Nonogram
        складність автоматично зростає:
        1–10 раунди → легкі (easy)
        11–20 раунди → середні (medium)
        21–30 раунди → складні (hard / expert)

 Це створює відчуття прогресії і “challenge mode”.

 Екрани застосунку (оновлена структура)
1. Екран “Поточний раунд” (Home / Round Hub)

Призначення: старт гри та вибір наступного раунду.

UI:

        “Поточний раунд: №X”
        кнопка “Почати / Продовжити”
        кнопка “Наступний раунд”
        індикатор складності (Easy / Medium / Hard)
        прогрес (наприклад: 7/30)

Дії:

        старт поточного раунду
        перехід до наступного
        перегляд прогресу
2. Екран гри (Game Screen)

Призначення: проходження Nonogram.

UI:

        ігрова сітка
        підказки (числа зверху і збоку)
        таймер
        кнопки:
        “Перевірити”
        “Фініш”
        “Пропустити раунд” (опційно після MVP)

Дії:

        заповнення клітинок
        перевірка правильності
        завершення раунду
3. Екран результату (Result Screen)

Призначення: показ результату раунду.

UI:

        час проходження
        % правильності
        кількість помилок
        оцінка (⭐ або “Perfect / Good / Try again”)
        кнопка “Далі”

Дії:

        перегляд результату
        перехід до наступного раунду
        Сценарій гри (основний flow)
        Відкривається “Поточний раунд”
        Користувач натискає “Почати”
        Відкривається Game Screen
        Гравець розв’язує Nonogram
        Натискає “Перевірити” або “Фініш”
        Відкривається Result Screen
Показ:
        час
        точність
        оцінка
        Натискає “Далі”
        Переходить до наступного раунду
Система складності (додана логіка прогресії)
Раунди	Складність
        1–10	Easy
        11–20	Medium
        21–30	Hard
        31+	Expert / Random chaos mode 

Можна додати бонус:

після 10 раунду — таймер стає швидший
після 20 — з’являються більші сітки (10x10 → 15x15 → 20x20)
 Оновлена архітектура
UI layer
        RoundHubScreen (новий головний екран)
        GameScreen
        ResultScreen
        DifficultyIndicatorWidget
        ProgressBarWidget
Logic layer
        RoundManager (керує раундами)
        DifficultyScaler (збільшує складність)
        GameEngine (перевірка Nonogram)
        TimerService
        ScoreCalculator (час + помилки → результат)
Data layer
        RoundRepository (зберігає прогрес раундів)
        PuzzleRepository (головоломки)
        ResultRepository (історія результатів)
        UserProgressRepository
 MVP (оновлений)
End-to-end сценарій MVP:

Раунд → гра → перевірка → результат → наступний раунд

MVP функції:
        10 перших раундів
        Nonogram поле
        перевірка правильності
        таймер
        результат (час + помилки)
        перехід між раундами
 Після MVP:
        30+ раундів і прогресія складності
        профіль користувача
        таблиця рекордів
        підказки
        пропуск раунду
        система оцінок (S/A/B/C)
        анімації

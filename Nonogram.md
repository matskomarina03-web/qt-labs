Практична робота №22
Тема: Індивідуальний проєкт. Проєктування інтерфейсу: екрани, навігація та стани UI
Мета:

        Сформувати практичні навички проєктування інтерфейсу застосунку на основі вимог і MVP, визначення набору екранів та їхніх станів, побудови навігаційної схеми між екранами, а також підготовки структури UI (макети, ключові елементи керування) для подальшого підключення логіки.

1. Визначення екранів MVP

На основі ПР21 для MVP застосунку Nonogram (раундова система гри) визначено 3 основні екрани:

Екран поточного раунду (Round Hub Screen) — стартова сторінка, де користувач бачить поточний раунд, прогрес і може почати або продовжити гру.
Екран гри (Game Screen) — основний екран проходження Nonogram-головоломки.
Екран результату (Result Screen) — відображає результат проходження раунду (час, точність, помилки).

Ці екрани реалізують повний MVP-сценарій:
поточний раунд → гра → результат → наступний раунд

2. Опис екранів
2.1. Екран поточного раунду (Round Hub Screen)
        Задача користувача:

        Розпочати або продовжити поточний раунд та перейти до наступного.

        Дані:
                номер поточного раунду
                рівень складності (Easy / Medium / Hard)
                прогрес (наприклад 7/30)
                статус незавершеного раунду
        Дії:
                почати / продовжити раунд
                перейти до наступного раунду
                переглянути прогрес гри
2.2. Екран гри (Game Screen)
Задача користувача:

Розв’язати Nonogram у межах конкретного раунду.

        Дані:
                ігрова сітка
                підказки (числа)
                таймер
                поточний стан заповнення
        Дії:
                заповнювати клітинки
                перевірити правильність рішення
                завершити раунд
                перезапустити раунд
3. Екран результату (Result Screen)
Задача користувача:

Переглянути результат проходження раунду.

        Дані:
                час проходження
                кількість помилок
                відсоток правильності
                оцінка (Perfect / Good / Fail)
        Дії:
                перейти до наступного раунду
                повторити раунд
                повернутися до списку раундів
3. Структура UI екранів
3.1 Round Hub Screen
Блоки:
        верхня панель (назва + прогрес)
        блок інформації про раунд
        блок вибору дії
        кнопка старту
        Центральна область:

інформація про поточний раунд + прогрес

3.2  Game Screen
Блоки:
        верхня панель (таймер + номер раунду)
        ігрова зона
        нижня панель команд
Елементи:
        сітка Nonogram
        кнопки: Check / Finish / Restart
Центральна область:

        ігрове поле (Nonogram)

3.3 Result Screen
Блоки:
        блок результатів
        блок оцінки
        панель навігації
Елементи:
        час проходження
        точність
        кнопки: Next / Retry / Back
Центральна область:

        підсумок раунду

4. Стани екранів
4.1. Round Hub Screen
        порожній стан: перший запуск, немає прогресу
        звичайний стан: показ поточного раунду
        завершення: всі раунди пройдені
        помилка: не вдалося завантажити прогрес
4.2. Game Screen
        активна гра: користувач заповнює сітку
        пауза/збережено: раунд збережено
        перевірка: обробка результату
        помилка: некоректні дані раунду
4.3. Result Screen
        стандартний результат: показ статистики
        незавершений раунд: гра не пройдена
        ідеальний результат: 100% правильності
        помилка завантаження результату
4.5. Навігаційна схема
Стартовий екран:

        Round Hub Screen

Переходи:
Round Hub → Game Screen
тригер: кнопка “Start / Continue”
Game Screen → Result Screen
тригер: “Finish” або завершення раунду
Result Screen → Game Screen
тригер: “Retry”
Result Screen → Round Hub
тригер: “Back” або “Next”
Game Screen → Round Hub
тригер: “Exit”
Повернення:
завжди через кнопку Back
прогрес раунду автоматично зберігається
6. Екрани vs діалоги

У застосунку Nonogram:

Екрани:
        Round Hub
        Game Screen
        Result Screen
Діалоги:
        підтвердження “Finish”
        підтвердження “Restart”
        підтвердження “Exit”
 Діалоги використовуються тільки для важливих дій, щоб уникнути випадкового завершення раунду.

7. Варфрейми (опис розташування)
Round Hub Screen
зверху: назва + прогрес (1/30)
центр: інформація про раунд
низ: кнопка Start / Continue
Game Screen
верх: таймер + раунд №
центр: Nonogram grid (головна зона)
низ: Check / Finish / Restart
Result Screen
центр: результат (час, точність, помилки)
низ: Next / Retry / Back
8. Узгоджені команди інтерфейсу
Основні команди:
Start / Continue — запуск раунду
Check — перевірка правильності
Finish — завершення раунду (з підтвердженням)
Restart — перезапуск раунду
Next — наступний раунд
Back — повернення назад
Правила доступності:
Check доступна тільки під час гри
Finish недоступна до старту раунду
Next доступна тільки після завершення
Restart потребує підтвердження
Continue доступна тільки якщо є збережений прогрес
9. Висновок

Інтерфейс застосунку побудований навколо раундової логіки Nonogram. Основні екрани забезпечують повний цикл взаємодії користувача: вибір раунду, проходження гри та отримання результату. Навігація проста та лінійна, що дозволяє легко підтримувати MVP і масштабувати застосунок у майбутньому.

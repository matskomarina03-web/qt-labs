1. На основі ПР21 для MVP застосунку Nonogram визначено 3 основні екрани. Перший — екран вибору складності та головоломки, де користувач обирає рівень гри та конкретний пазл. Другий — екран гри, який є основним і містить ігрове поле, таймер і кнопки перевірки та завершення. Третій — екран результату, де відображається час проходження, кількість помилок і статус гри. Ці екрани повністю покривають головний сценарій MVP: вибір головоломки → проходження → отримання результату.
2. 
        1. Вибір складності та головоломки — вибір рівня гри

        Задача користувача: обрати складність і конкретну головоломку для проходження.
        Дані: список рівнів (easy/medium/hard), список puzzle, статус прогресу.
        Дії: вибрати складність, відкрити головоломку, продовжити незавершену гру.

        2. Екран гри — проходження Nonogram

        Задача користувача: розв’язати головоломку.
        Дані: ігрова сітка, підказки, таймер, поточний стан гри.
        Дії: заповнювати клітинки, перевірити рішення, завершити гру, почати заново.

        3. Екран результату — перегляд підсумку

        Задача користувача: переглянути результат проходження.
        Дані: час, кількість помилок, статус (пройдено/не пройдено).
        Дії: повернутися до вибору рівнів, повторити рівень, переглянути статистику.

3. 
        1. Екран вибору складності та головоломки

        Головні блоки інтерфейсу:

        верхня панель (назва, прогрес користувача)
        блок вибору складності
        список головоломок
        панель навігації

        Ключові елементи керування:

        кнопки вибору рівня (Easy / Medium / Hard)
        список/картки головоломок
        кнопка “Continue” (якщо є незавершена гра)

        Центральна область:
        список головоломок для вибраного рівня

        2. Екран гри (GameScreen)

        Головні блоки інтерфейсу:

        верхня панель (таймер, статус гри)
        ігрова область
        нижня панель команд

        Ключові елементи керування:

        ігрова сітка Nonogram
        кнопки “Check”, “Finish”, “Restart”
        індикатор помилок/стану

        Центральна область:
        ігрове поле (сітка Nonogram)

        3. Екран результату

        Головні блоки інтерфейсу:

        блок загальної статистики
        блок статусу проходження
        панель навігації

        Ключові елементи керування:

        кнопка “Back”
        кнопка “Retry”
        відображення статистики (час, помилки)

        Центральна область:
        результат проходження (час, помилки, статус)
4. 
        1. Екран вибору складності та головоломки
        Порожній стан: немає доступних головоломок або перший запуск застосунку
        Звичайний стан: відображається список рівнів і доступні головоломки
        Стан прогресу: частина головоломок позначена як пройдені або незавершені
        Помилка: не вдалося завантажити список головоломок
        2. Екран гри (GameScreen)
        Активна гра: користувач заповнює сітку, таймер працює
        Пауза / збережений стан: гра зупинена або відновлена після виходу
        Редагування (ігровий процес): користувач змінює клітинки (основний режим гри)
        Помилка: некоректні дані головоломки або збій перевірки
        3. Екран результату
        Звичайний стан: показано результат гри (час, помилки, статус)
        Порожній стан: немає збереженого результату (наприклад, перший запуск або вихід без завершення)
        Стан незавершеної гри: відображається, що рівень не пройдено до кінця
        Помилка: не вдалося завантажити результат із пам’яті

5. Стартовий екран
        PuzzleListScreen (вибір головоломки) після запуску/входу
        Переходи
        PuzzleList → GameScreen
        тригер: вибір головоломки / кнопка “Continue”
        GameScreen → ResultScreen
        тригер: кнопка “Finish” або завершення гри
        GameScreen → PuzzleList
        тригер: кнопка “Back/Exit”
        ResultScreen → PuzzleList
        тригер: кнопка “Back” або “Retry”
        Повернення
        через кнопку Back на кожному екрані
        прогрес у грі зберігається при виході з GameScreen

6.  У застосунку Nonogram не всі дії повинні відкривати новий екран. Частина дій виконується у вигляді діалогових вікон, щоб не перевантажувати навігацію.

Окремими екранами є всі основні частини гри: вибір головоломки, сама гра та екран результату. Саме між ними користувач рухається під час роботи із застосунком.

Діалоги використовуються тільки для підтвердження важливих дій. Наприклад, якщо користувач натискає “Finish”, відкривається вікно підтвердження, щоб випадково не завершити гру. Так само підтвердження потрібно для перезапуску рівня або виходу без збереження.

Усі інші дії залишаються в межах одного екрана. Наприклад, заповнення клітинок у грі, перевірка результату або зміна складності не відкривають нових вікон, а просто змінюють стан поточного екрана.

7. 
        Екран вибору головоломки

        На цьому екрані зверху розташовується вибір складності (Easy, Medium, Hard). Під ним знаходиться список головоломок, які змінюються залежно від обраного рівня. Кожна головоломка показується як окремий елемент списку зі статусом (нова, пройдена або незавершена). Внизу екрана може бути кнопка продовження гри, якщо користувач уже почав рівень раніше.
        Центральною частиною цього екрана є саме список головоломок, оскільки саме тут відбувається вибір гри.

        Екран гри

        У верхній частині екрана знаходиться таймер і коротка інформація про прогрес. Основну частину займає ігрове поле Nonogram — це велика сітка, з якою взаємодіє користувач.
        У нижній частині розташовані кнопки керування: перевірка, завершення гри та перезапуск. Під час гри ці кнопки можуть змінювати стан, наприклад ставати неактивними або відкривати діалог підтвердження.
        Центральною областю є ігрова сітка, тому що саме вона є основною дією користувача.

        Екран результату

        На цьому екрані в центрі показується результат проходження головоломки. Це може бути час, кількість помилок і статус (чи пройдено рівень). Нижче розташовані кнопки повернення або повторної спроби.
        Цей екран є простим, оскільки він тільки показує підсумок і не містить складної взаємодії.

8. У всьому застосунку використовуються однакові команди, щоб користувач не плутався.

Кнопка “Back” завжди означає повернення назад на попередній екран.
Кнопка “Finish” завжди завершує гру, але перед цим обов’язково відкривається підтвердження.
Кнопка “Restart” перезапускає рівень, але також потребує підтвердження.
Кнопка “Check” використовується тільки під час гри для перевірки правильності рішення.
Кнопка “Continue” з’являється тільки якщо є незавершена гра.

Важливо, що ці кнопки не з’являються хаотично — вони залежать від стану гри. Наприклад, “Continue” недоступна, якщо користувач ще не починав жодного рівня, а “Check” неактивна після завершення гри.
